Lesson 2.1: Introduction to "SERIOUS" Programming

A Computer

Referencing back to stage 1 we mentioned computers are stupid. Well, they really are. A computer will just sit there until we tell it what to do. A computer can be programmed to do anything we want. All we have to do is write a properly structured program and run it.

A Computer Program
A program comes in many forms and uses. Actually, as soon as you turn on any computer or device you are running some kind of program. Email, web browsing, browser and mobile app are all programs.

Programming Language
This language is what is used to tell a computer what to do. The programming language was developed to eliminate language ambiguity. From person to persons and country to country things we say could have different meaning. Regular language interpretation could be an issue for computing. The language we are going to use is Python.

Python
Python is a programming language interpreter. When you write your code Python converts that code to instruction a computer can act on. Sounds easy right? Well, keeping in mind your computer is stupid you must use proper grammar. When using Python you must use the exact code in order for the interpreter to run the code. Backus-Naur Form describes python language rule. It is a formal, mathematical way to specify context-free grammars that is both precise and unambiguous.
A fun little article can be found here.

Expressions
Python interpreter is a language and only knows how to evaluate code that is part of the python language. If something is written that isn't part of the language it will generate a syntax error. Python grammar uses expressions. Python grammar uses expressions. Replacement grammar for arithmetic expression structure.

Expression -> Expression - Operator - Expression

Expression = number
Operator = +, *, - etc.
Numbers = 1, 2, 3 etc.

What a function is?
A function is programmer defined. You give a function a set of inputs, You tell the function what to do with it and the function outputs your result.

How to make a function/procedure:
To define or create your function you first def and name your function. Name is usually related to what the function is doing. Some procedures are built in to python and will save you some time. It is possible to build or define your own procedure to get your computer to do what you want it to do. You would then use a block statement to tell your function what you want it to do with the input information.

How to use a function/procedure:
You first need to define your procedure and give it some inputs. def procedure(input, input, input, etc.) as long as the number of inputs matches what's expected. You can also set variables for your inputs. This would allow you to use the same function with different information without rewriting the function. We can also compose a function by using the output of one function as the input for another. It's a good idea that the names of your input relates to what it is.

Functions defined vs. used
When defining a function you give it a name and inputs. You then set the value of what you want to do or what you want it to return. Return produces /defines what your output will be.
To call or use a function you will need the function name (the value of the parameter/input) preceded by print

Lesson 2.5: Debugging

Debugging

Bugs Happen!
WOW! You can say that again. Bugs are inherent to coding. It's just going to happen and rather than banging your head on the wall trying to figure out how to avoid it maybe, try developing a strategy for systematically getting rid of them.

Syntax Error
Python sometimes give error messages when the code crashes. This error- code is called Traceback and identifies what line in your code it crashed on. Sometimes bugs don't actually crash your code but calculates an incorrect value it can be difficult to locate. Introducing print statements within your code can help where things went wrong.

Example
Print "# DEBUG:"
Print "# location", location

Comparing Code
Comparing your crashing code to a known working similar code is a huge helper in debugging. You can “#” your code out and see how the know working code is working and slowly add your code back in making small changes until it works.

5 debugging strategies
  1. Examine error messages
  2. Work from example code
  3. Make sure examples work
  4. Check (print) intermediate results
  5. Keep and compare old versions of your code

Lesson 2.6: Structured Data: Lists & for Loops

Strings vs. Lists
With strings all the elements have to be have to be characters. With lists the elements can be whatever you want. Characters, Strings, Numbers, Other lists Strings and lists can both be assigned to a variable: a=’yabba!’ or p=[‘y’, ‘a’, ‘b’, ‘b’, ‘a’] You can use the square brackets to select elements. s[0] is ‘y’ or p[0] is ‘y’ Select a substring s[2:4] is ‘bb’ Remember with the 4 it’s that number -1. p[2:4] is [‘b’, ‘b’] <list_name> [ ] is called an empty list <list_name> [3] single element list and so on.

Nested Lists
Lists in lists: Searching for elements in this situation you would us more than one index. The first index identifies which list you are looking at and the second index is the element in that list. The number if index’s you use depends on how deep the nested lists go.

Mutation
Lists support mutation. This means that you can change the value of a list after you've created it.

How Strings Change
s = 'Hello'
s = 'Yello' In this case we didn’t modify the string ‘Hello’ we just changed the assignment of s
s = s + w
Once we concatenate this it takes string 'Yello' and string ‘w’ and creates a new string 'Yellow' "s" now refers to the new string 'Yellow'

How lists change
P=['H', 'e', 'l', 'l', 'o']
P[0] = ‘Y’ this changes the value of the list to p=['Y', 'e', 'l', 'l', 'o']
The key difference between mutable objects and non-mutable objects is once an object is mutable we have to worry about other variables that might refer to the same object.

List Operations
.append is like a procedure but it's a method. Append will add a new element to the end of a list. List .append (new element) is List new element.
Important thought here is that append is mutating the list, not creating a new list.

+ This is similar to the concatenation in that it adds lists together.
[0,1] + [2,3] is [0,1,2,3]
This produces a new list

len Stands for length. We pass into “len” the object we want to know the length of. Len works for lists, strings or anything that’s a collection of things. The output of len is the number of elements in the input.

Loops on lists
While loops evaluate to true or false. While evaluates it’s test expression and if true, executes the block statement and will continue to do so as long as the test expression is true.

for Loops Python has provides a more simple way to loop through elements of a lists.
This is similar to the structure of the while loop and if statement.
Structure:

for <variable_name> in <list_name>:
<block>

find_element vs. .index
find_element takes two inputs. The first input is a list of anything. The second input can be a value of any type. You are searching in your list for that value.
Structure:
find_element ([2, 4, 6] 2) The 2 is in position [0] so the output >>> 0
find_element (['to', 'too', 'two'] 'three') There isn't match so the output >>>-1 but only if you define the -1

Another way to define find_element is with .index
Structure: print <list>.index(<value>) or print p.index(5).
This will search the assigned p list for 5 or whatever you are looking for. If what you are looking for isn't in the list it will error.

Another list operation that will be useful. in In: <value> in <list>
This actually makes more sense because it relates to the way we talk. "Is this value in this list?" With in if the value is in the list the output is True. If it isn’t the output is False. “not in” is just the opposite.
